## 📝 Lessons Learned

**Cleanup:**
- Archive old files regularly (weekly)
- Keep active designs separate from archive
- Delete test/temp files immediately
- Document as you go (don't wait)

**Documentation:**
- Keep it short, structured, useful
- Include workarounds for known issues
- List recommendations for tool improvements
- Make it living (update constantly)

**Organization:**
- Separate source code from config from docs
- Use clear folder names (src, config, docs, designs)
- Archive old designs, don't delete (historical value)
- Keep .env out of git (add to .gitignore)

## 🧠 Marketing Optimization System (Option C)

**Approach:** Track Everything → Weekly Analysis → Recommend → Test → Iterate

**Created:** `docs/MARKETING-STRATEGY.md` — Living strategy document

### Phases

**Phase 1: Data Collection (Weeks 1-4)**
- Gather baseline analytics for all posts
- Document hypotheses for each post
- Track: promotion types, times, styles, headlines, visuals
- No major changes — establish patterns

**Phase 2: Pattern Recognition (Week 4-5)**
- Analyze 4 weeks of data
- Identify top/bottom performers
- Surface clear patterns
- Form initial recommendations

**Phase 3: Optimization (Week 6+)**
- Implement data-driven changes
- A/B test specific variables
- Weekly iteration based on results
- Continuous improvement loop

### Variables We Test

| Variable | Options | Initial Hypothesis |
|----------|---------|-------------------|
| **Promotion Type** | BOGO, % off, $ off, Free item | BOGO performs best |
| **Posting Time** | Morning (9-11), Afternoon (2-4), Evening (6-8) | Afternoon best |
| **Headline Style** | Urgent, Fun, Informative, FOMO | Urgent + emoji best |
| **Visual Focus** | Food-only, People + food, Text-heavy | Food-focused best |
| **Caption Length** | Short (<100), Medium (100-200), Long (200+) | Medium best |
| **CTA Type** | Direct, Casual, Question, FOMO | FOMO best |
